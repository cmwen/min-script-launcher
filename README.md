# min-script-launcher

>

> integration, and git-backed versioning.

Native CLI/TUI for finding and running scripts generated by coding agents.

The launcher scans fixed script folders such as `~/bin` and `~/.local/bin`, lets you search by filename, metadata, tags, usage, aliases, and top-of-file comments, previews the command, then executes the selected script from the directory where you launched `msl`.

## Status

This project targets macOS and Linux. It uses TypeScript, Ink, React, and Bun native executable builds.

```bash
bun build ./packages/cli/src/index.ts --compile --outfile ./packages/cli/dist/msl
```

## Usage

```bash
# Open the TUI picker
msl

# Search without opening the TUI
msl search repo cleanup

# List all discovered scripts
msl list

# Run by command name, display name, alias, or absolute path
msl run repo-status -- --json

# Preview the exact command
msl run repo-status --dry-run -- --json

# Print metadata guidance for coding agents
msl guide repo-status
```

By default, `msl` scans:

- `~/bin`
- `~/.local/bin`

Customize the catalog with repeated flags or an environment variable:

```bash
msl --dir ~/dotfiles/bin --dir ~/.local/bin
MSL_SCRIPT_DIRS="$HOME/bin:$HOME/.local/bin:$HOME/dotfiles/bin" msl
```

## Script Metadata

Scripts work without metadata, but generated scripts become much easier to find when agents add `msl:*` comments near the top:

```bash
#!/usr/bin/env bash
# msl:name Repo Status
# msl:description Summarize git status, branches, and recent commits for the current repo.
# msl:usage repo-status [--json]
# msl:tags git, repo, status
# msl:alias rs
# msl:example repo-status --json

set -euo pipefail
```

Scripts should be executable and should treat the process current working directory as their operating context. That lets a script live in a fixed folder while acting on whichever project you ran `msl` from.

Broken symlinks and unreadable directories are skipped with warnings instead of crashing the catalog.

## Web And MCP

The web UI is search-only by design:

```bash
msl web --port 3000
```

The MCP server exposes script search, script explanation, and metadata guidance:

```bash
msl mcp
```

## Development

```bash
# install dependencies
bun install

# build TypeScript packages and the web client
bun run build

# run tests
bun run test

# format and lint
bun run lint
```

If Bun is installed, build the native CLI with:

```bash
bun run build:native:current
```

Build all release bundles with:

```bash
bun run build:native
```

That writes archives and checksums to `dist/release`:

- `msl-linux-x64.tar.gz`
- `msl-linux-arm64.tar.gz`
- `msl-darwin-x64.tar.gz`
- `msl-darwin-arm64.tar.gz`
- `SHA256SUMS`

Version tags matching `v*` run the release workflow. The workflow validates the project, builds those native bundles, and attaches them to the GitHub release.
